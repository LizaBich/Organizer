# Organizer
# Требования к проекту

# 1 Введение
Цель разработки – приложение для десктопных устройств под управлением операционной системы (ОС) Windows. Предназначено для 
хранения активностей пользователя, таких как события календаря, а также заметок.

# 2 Требования пользователя
# 2.1 Программные интерфейсы
Приложение будет написано на языке C# в "Visual Studio 2017" с использованием Windows Presentation Foundation (WPF) для интерфейса пользователя и EntityFramework для построения базы данных.
Аналогами являются такие приложения, как "Google Calendar" и "Google Keep". Данные приложения предоставляют ряд возможностей. Так "Google Calendar" позволяет добавлять события. Данное приложение 
обеспечивает гибкую систему настройки повтора события. Также есть возможность посылать уведомления по email. "Google Keep" позволяет добавлять заметки. Имеется возможность менять цвет заметки, 
добавлять вложения.

# 2.3 Характеристика пользователя
Целевой аудиторией данного приложения являются любые пользователи, желающие иметь список заметок и событий на своём ПК.
Минимальный необходимый навык – умение пользоваться персональным компьютером под управлением ОС Windows.

# 2.4 Предположения и зависимости
С целью оптимизации работы приложения и пользователя, последнему предоставлена возможность создать
максимум 50 заметок. 

# 3 Системные требования
Любое устройство под управлением Windows 10.
# 3.1 Функциональные требования
1. Возможность добавить событие на любой день.
2. Возможность просмотреть событие.
3. Возможность изменить событие.
4. Возможность удалить событие.
5. Возможность добавить заметку. 
6. Возможность просмотреть заметку.
7. Возможность изменить заметку.
8. Возможность удалить заметку.
9. Возможность поиска заметок в списке по имени.

# 3.2 Нефункциональные требования
# 3.2.1 Атрибуты качества
Экономия памяти (не более 50 Мб). Расширение базы данных увеличивает потребление памяти устройства, следовательно, необходимо предусмотреть экономное её использование приложением.
